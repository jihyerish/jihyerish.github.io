맞춤형 개발 플랜

현재 프로파일 기준 · TPdM 우선 경로 추천 플랜

🎯
추천 경로: TPdM (Technical Product Manager)
브라우저 엔지니어링 깊이 + VC 투자 경험의 조합은 제품 전략 역할에서 훨씬 강력한 차별화를 만듭니다. TPdM은 "무엇을 만들 것인가"를 정의하는 역할로, 기술적 판단력과 비즈니스 감각이 모두 요구됩니다.
01
기반 확립 단계
2026년 하반기 – 2027년
1-2년차
핵심 실행 과제
  • 📋
    제품 디스커버리 프레임워크 내재화

    JTBD(Jobs-to-be-Done), 사용자 인터뷰 설계, PRD 작성 역량을 실무에서 즉시 적용. Figma 기반 와이어프레임 및 제품 스펙 문서 작성 루틴 확립.

  • 📊
    데이터 기반 의사결정 시스템 구축

    SQL 기초 + 제품 분석 툴(Amplitude, Mixpanel) 습득. 노스스타 메트릭 정의와 대시보드 구성. A/B 테스트 설계와 통계적 유의성 해석.

  • 🎓
    CSPO 취득 (2026 하반기 목표)

    현재 스프린트 관리 경험을 공식화. 백로그 우선순위화 프레임워크(RICE, MoSCoW)를 제품 업무에 즉시 적용.

브라우저 전문성 레버리지
  • 🌐
    웹 플랫폼 제품 특화 포지셔닝

    브라우저/웹 표준 기반 제품 (PWA, WebAssembly, WebRTC 기반 서비스)의 PM으로 포지셔닝. 기술 깊이를 제품 차별화 요소로 마케팅.

  • 🔗
    오픈소스 기여를 포트폴리오화

    Igalia/Chromium 기여 이력을 제품 사례로 재구성. "기술적 제약을 이해하는 PM"으로 브랜딩.
